## Ticket Filing Rules

- Each ticket must link this spec and the specific artifacts it depends on.
- Tickets that touch public/auth/mobile/persistence/cutover behavior must include contract tests before implementation is considered complete.
- Tickets that implement public-edge proof must include fail-closed tests for missing/invalid/revoked phone proof, missing/invalid/revoked node proof, origin bypass without edge assertion, route allowlist denial, replayed proof, and LAN-first node fallback.
- Tickets must follow [cutover_scope.md](cutover_scope.md). Removed surfaces must not be reintroduced without a later owner-approved issue.
- Tickets that write durable stores must state ownership, backup, and rollback behavior.
- Tickets that retain Python CLI behavior must state whether the command is HTTP-only, read-only local, CLI-owned writer, Rust-routed writer, or retired.
- Mobile app and on-the-go attach regressions block first Rust release tickets.
- A Python compatibility shim is acceptable only when it has an explicit retirement gate and does not hide state ownership ambiguity.
