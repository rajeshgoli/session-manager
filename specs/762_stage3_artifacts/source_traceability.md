# Stage 3 Source Traceability

Status: converged after three sequential independent reviewer convergence signals.

This artifact links the Stage 3 behavior matrices to source and tests. It is not a replacement for the main spec; it is the audit trail for high-risk internal contracts that later Rust implementation tickets must preserve or explicitly change through Stage 4/5 review.

## Reconciliation Notes

| Finding / Risk | Resolution |
|----------------|------------|
| Native mobile behavior beyond terminal attach was under-specified. | Main spec now includes native mobile auth/bootstrap/status/analytics rows and test candidates. |
| App artifact deployment/storage internals were omitted. | Main spec now includes artifact publish/serve ordering, metadata, cache headers, and test candidates. |
| Codex structured request startup reconciliation was omitted. | Main spec now includes process-generation orphaning and migration-drain requirement. |
| Durable review/job watch notification ordering was under-specified. | Main spec now preserves queue-before-terminal-watch-persist ordering and crash-window tests. |
| Stage 3 lacked ordered recovery and explicit state-transition handoff artifacts. | Added [ordered recovery](ordered_recovery.md) and [state transitions](state_transitions.md). |
| Termux attach command/fallback semantics were under-specified. | Main spec and this artifact now record Termux SSH script generation, retry, and LAN fallback as current Python history only; owner has deprecated it and Rust must not port it. |
| App artifact tests are cited but not executable in the current declared dependency set. | App artifact row now records the `python-multipart` fixture dependency caveat; behavior contract remains source-derived. |
